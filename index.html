    My First Page: Route, Controller, Template

    /Resources/config/routing.yml

    Default Route

                my_company_my_first_homepage:
            path:     /hello/{name}
            defaults: { _controller: MyCompanyMyFirstBundle:Default:index }

    Line #1 is the name of the route.

    This is what you will use to call the route from inside your project.

    Line #2 is the path.

    Words in curly brackets {} are variables.

    Line #3 is the definition.

    This specifies the specific controller file (Default) and function (index) that this route will call. The controller name may optionally be followed with a comma and the default values for route variables:

                index:

    Default Controller

    Controller file names end with the word Controller.

    Controller functions (the logic for an individual page) end with the word Action.

    So the Default controller is named DefaultController and the default controller function, index, is named indexAction.

                namespace MyCompany\MyFirstBundle\Controller;

                }
            }

    Default Template

    The .twig extension on our Templates means we are able to use Twig tags to render our Template.

    To simply display a variable with Twig, surround the variable name with double curly brackets {{}}.

                    Hello {{ name }}!

      Twig will execute simple logic like if/else, for loops, and variable assignments when surrounded by {% %}

      • Example: {% set condition = 'potato' %}
      • Example: {% if condition == 'yes' %}Conditional Content{% endif %}
      • Reference: http://twig.sensiolabs.org/

                        public function selectItemAction()
                    {
                        $em = $this->getDoctrine()->getManager();

                        $items = $em->getRepository('MyCompanyMyFirstProject:Item')->findAll();

                        $query = $em->createQueryBuilder()
                            ->select('c')
                            ->from('MyCompanyMyFirstProject:Item', 'c')
                            ->where('c.name LIKE :prefer')
                            ->setParameters(array('prefer' => 'Armor'))
                            ->setMaxResults(1)
                            ->orderBy("c.value","DESC")
                            ->getQuery();

                        $recommend = $query->getSingleResult();

    Creating a Service

    Create a new directory:

  • src/MyCompany/MyFirstProject/Services

    • Create a new Php file:

  • src/MyCompany/MyFirstProject/Services/CharacterEffect.php

    • Define the new Service:

                    parameters:
                    my.service.location:      MyCompany\MyFirstProject\Services\

                services:
                    character_effect:
                        class:                  %my.service.location%CharacterEffect
                        arguments:
                            entityManager:      "@doctrine.orm.entity_manager"

    Creating a Service

                    namespace MyCompany\MyFirstProject\Services;
                use Doctrine\ORM\EntityManager;
                use MyCompany\MyFirstProject\Entity\Character;

                class CharacterEffect
                {
                    public function __construct(EntityManager $entityManager)
                    {
                        $this->em = $entityManager;
                    }

                    public function levelUp(Character $character)
                    {
                        $em = $this->em;

                        $name = $character->getName();
                        $new_level = $character->getLevel() + 1;

                        $character->setLevel($new_level);
                        $em->persist($character);
                        $em->flush();

                        $message = "$name has reached level $new_level!";

                        return $message;
                    }
